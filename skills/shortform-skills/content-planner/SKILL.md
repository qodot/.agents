# 숏폼 마케팅 컨텐츠 기획서 생성기

이커머스 제품에 대한 인플루언서 마케팅용 숏폼 컨텐츠 기획서를 체계적으로 생성하는 스킬입니다.

## 의존 도구 및 스킬

이 스킬은 다음 도구/스킬들과 함께 사용합니다:

**필수 확장:**
- `ask`: 사용자에게 객관식 질문 (단일)
- `ask_multi`: 사용자에게 여러 객관식 질문 (순차적)

**권장 스킬:**
- `browser-tools`: PDP 페이지 조회, 브랜드 조사, 트렌드 분석 (필수 권장)
- `youtube-transcript`: 인기 숏폼 컨텐츠 분석

**연계 스킬 (shortform-skills 내):**
- `hook-planner`: 첫 3초 후킹 기획 (7단계에서 사용)
- `creator-planner`: 크리에이터 선정 기획 (기획서 완료 후 사용)

## 프로세스

### 1단계: 기본 정보 수집

사용자에게 다음 **2가지만** 먼저 수집합니다.

> 💡 **ask 확장 사용**: 브랜드명은 `ask` 도구로 수집하고, PDP URL은 직접 입력을 요청합니다.

```
📦 기본 정보 (필수)
1. 브랜드명:
2. PDP URL: (메인 제품 상세 페이지)
```

### 2단계: 브랜드 및 제품 자동 조사

> 💡 **browser-tools 스킬을 사용하여 브랜드 공식 사이트와 PDP 페이지를 조회합니다.**

**브랜드 조사** (browser-tools 사용):
```bash
# 브랜드 공식 사이트 방문
{browser-tools-dir}/browser-nav.js [브랜드 공식 사이트 URL]
{browser-tools-dir}/browser-screenshot.js

# About/브랜드 스토리 페이지 확인
# 페이지 끝까지 스크롤하며 브랜드 정보 수집
```
- 브랜드 카테고리, 포지셔닝, 타겟층 추론

**URL에서 제품 정보 추출** (browser-tools 스킬 사용):

> ⚠️ **중요: PDP 페이지 조회 시 반드시 browser-tools 스킬을 사용합니다.**
> 
> 많은 이커머스 사이트는 JavaScript로 동적 렌더링되므로, 단순 HTTP 요청으로는 제품 정보를 추출할 수 없습니다.

> 🔴 **필수: 페이지 끝까지 스크롤하여 모든 정보를 확인해야 합니다.**
> 
> PDP 페이지에는 제품 상세 정보, 성분, 사용법, 리뷰 등이 페이지 하단에 위치하는 경우가 많습니다.
> **중간에 멈추지 말고 페이지 끝까지 스크롤하여 모든 정보를 수집하세요.**

```bash
# 1. 브라우저로 PDP 페이지 이동
{browser-tools-dir}/browser-nav.js [PDP URL]

# 2. 스크린샷으로 상단 확인
{browser-tools-dir}/browser-screenshot.js

# 3. 제품상세 탭이 있으면 클릭
{browser-tools-dir}/browser-eval.js '(() => { /* 제품상세 탭 클릭 */ })()'

# 4. 페이지 끝까지 스크롤하며 모든 정보 수집 (필수!)
#    - 800px씩 스크롤 → 스크린샷 → 반복
#    - 페이지 끝에 도달할 때까지 계속
{browser-tools-dir}/browser-eval.js 'window.scrollBy(0, 800)'
{browser-tools-dir}/browser-screenshot.js
# ... 페이지 끝까지 반복 ...
```

**PDP 조회 체크리스트:**
- [ ] 제품명, 가격 확인
- [ ] 제품 상세 이미지 전체 확인
- [ ] 성분/소재/기술 스펙 확인
- [ ] 사용 방법/착용 방법 확인
- [ ] 브랜드 스토리/USP 확인
- [ ] 리뷰/평점 확인 (있는 경우)
- [ ] **페이지 끝까지 스크롤 완료**

**추출해야 할 정보:**
- 제품명, 가격, 주요 성분/특징, USP 파악
- 제품 카테고리 추론
- 제품 이미지/비주얼 스타일
- 브랜드 톤앤매너

### 3단계: 추론 결과 확인

조사 결과를 바탕으로 **추론한 내용을 사용자에게 확인**합니다.

> 💡 **ask 확장 사용**: 분석 결과를 표로 보여준 후, `ask` 도구로 확인을 받습니다.
> - 질문: "분석 결과가 맞나요?"
> - 옵션: ["맞습니다", "수정이 필요합니다"]

```
🔍 분석 결과 확인

제가 파악한 내용이 맞는지 확인해주세요:

| 항목 | 추론 결과 |
|------|----------|
| 브랜드 카테고리 | {화장품/F&B/패션/테크 등} |
| 제품 유형 | {세럼/크림/음료/의류 등} |
| 예상 타겟층 | {20-30대 여성, 스킨케어에 관심 등} |
| 제품 USP | {주요 차별점} |

👉 수정할 내용이 있으면 알려주세요!
```

### 4단계: 캠페인 정보 수집

추론이 확인되면 나머지 정보를 수집합니다.

> 💡 **ask_multi 확장 사용**: 다음 질문들을 `ask_multi` 도구로 순차적으로 수집합니다.
> ```
> questions: [
>   { key: "country", question: "대상 국가는?", options: ["한국", "미국", "일본", "동남아"] },
>   { key: "goal", question: "캠페인 목표는?", options: ["인지도 향상", "전환 유도", "브랜드 이미지 구축", "신규 고객 확보"] },
>   { key: "platform", question: "주요 플랫폼은?", options: ["TikTok", "Instagram Reels", "YouTube Shorts"] },
>   { key: "product_provision", question: "제품 제공 여부는?", options: ["제공", "미제공"] }
> ]
> ```
> - 타겟 오디언스와 리워드 조건은 직접 입력을 요청합니다.

```
📋 캠페인 정보

1. 대상 국가: (한국 / 미국 / 일본 / 동남아 / 기타)
2. 캠페인 목표: (인지도 향상 / 전환 유도 / 브랜드 이미지 구축 / 신규 고객 확보)
3. 주요 플랫폼: (TikTok / Instagram Reels / YouTube Shorts / 복수 선택)
4. 제품 제공 여부: (제공 / 미제공)
5. 리워드 조건: (금액, 지급 방식)
```

## 5단계: 트렌드 분석 (최근 3개월)

**browser-tools로 트렌드 조사**:

```bash
# TikTok/Instagram에서 해시태그 검색
{browser-tools-dir}/browser-nav.js "https://www.tiktok.com/search?q=[카테고리]+[국가]"
{browser-tools-dir}/browser-screenshot.js
# 인기 영상들의 조회수, 좋아요, 후킹 스타일 확인

# 경쟁 브랜드 공식 TikTok/Instagram 확인
{browser-tools-dir}/browser-nav.js "https://www.tiktok.com/@[경쟁브랜드]"
{browser-tools-dir}/browser-screenshot.js
# 최근 컨텐츠 스타일, 참여율 확인

# 트렌드 분석 사이트 활용 (선택)
# - TikTok Creative Center: https://ads.tiktok.com/business/creativecenter
# - Instagram Trends 등
```

**분석 항목**:
- 현재 인기 있는 컨텐츠 포맷
- 트렌딩 해시태그
- 바이럴 훅/밈
- 경쟁 브랜드 캠페인 사례
- 플랫폼별 알고리즘 변화

**충분한 분석 기준**:
- 최소 5개 이상의 관련 트렌드 파악
- 3개 이상의 경쟁/참고 사례 수집
- 플랫폼별 최신 알고리즘 특성 확인

| `[에스쁘아] 2차 - 뉴 비벨벳 커버쿠션.pdf` | 화장품 (색조) | - | 2차 수정본 |
| `[불스원] 2차 기획안.pdf` | 자동차용품 | - | 비화장품 카테고리 참고 |

### 참고 방법

기획서는 `../examples/` 폴더에 위치합니다.

1. **카테고리 매칭**: 현재 작업 중인 브랜드와 유사한 카테고리의 기획서 1-2개 선택
2. **PDF 읽기**: `read` 도구로 해당 PDF 파일 확인
   ```
   ../examples/[브랜드명] 기획안.pdf
   ```
3. **참고 포인트**:
   - Campaign Summary 표 구성 방식
   - 콘텐츠 상세 기획의 톤앤매너
   - 핵심 포인트 작성 스타일
   - 영상 흐름 구성 방식
   - 해시태그 전략

> ⚠️ **주의**: 기존 기획서는 **형식과 품질 수준 참고용**입니다. 컨텐츠 아이디어나 기획 내용을 그대로 복사하지 마세요. 반드시 현재 브랜드/제품에 맞는 새로운 기획을 작성해야 합니다.

## 7단계: 컨텐츠 전략 수립

수집된 정보와 트렌드 분석을 바탕으로 결정:

- **훅(Hook)**: 첫 3초 내 시청자를 사로잡을 요소 → **hook-planner 스킬 사용**
- **핵심 메시지**: 전달할 1가지 명확한 메시지
- **CTA**: 시청 후 유도할 행동
- **컨텐츠 포맷**: 튜토리얼 / 리뷰 / 챌린지 / 일상 브이로그 / 비포애프터 (트렌드 반영)

### 컨텐츠 훅(Hook) 기획

> ⚠️ **중요: hook-planner 스킬 사용 필수**
>
> 첫 3초 후킹 기획은 반드시 `hook-planner` 스킬을 사용합니다.
> 앞서 수집한 정보(브랜드, 제품, 타겟, 캠페인)를 활용하여 스킬을 실행합니다.
>
> ```
> /skill:hook-planner
> ```
>
> **hook-planner에서 수집하는 정보:**
> - 제품 정보: 카테고리, 가격대, USP, 제한사항
> - 타겟 정보: 국가, 연령대, 성별, 페인포인트
> - 캠페인 정보: 플랫폼, 목적, 구매 단계, 시즌
>
> 이 정보들은 4단계에서 이미 수집했으므로, hook-planner 실행 시 해당 정보를 전달합니다.
> hook-planner가 생성한 5가지 후킹 중 최적의 후킹을 선택하여 각 컨텐츠에 적용합니다.

## 8단계: 기획서 작성

[templates/content-plan-template.md](templates/content-plan-template.md) 템플릿을 기반으로 기획서를 작성합니다.

> ⚠️ **중요: 예시 기획 복사 금지**
> 
> 템플릿의 예시는 **형식과 톤앤매너 참고용**입니다. 컨텐츠명, 기획 내용, 핵심 포인트, 영상 흐름 등을 그대로 복사하거나 살짝만 변형하여 사용하는 것은 **절대 금지**합니다. 반드시 트렌드 분석 결과와 해당 브랜드/제품 특성에 맞는 **새로운 기획**을 작성해야 합니다.

## 품질 보증 체크리스트

기획서 작성 완료 후 다음 항목을 확인합니다:

- [ ] 브랜드 검색과 PDP 분석이 완료되었는가?
- [ ] 추론한 카테고리/타겟층이 사용자에게 확인되었는가?
- [ ] 최근 3개월 트렌드 분석이 충분히 수행되었는가? (5개+ 트렌드, 3개+ 사례)
- [ ] **유사 카테고리의 기존 기획서를 참고했는가?**
- [ ] Campaign Summary 표에 3개 이상의 컨텐츠가 정리되었는가?
- [ ] 각 콘텐츠 상세 기획에 크리에이터 유형이 명시되었는가?
- [ ] 각 콘텐츠에 핵심 포인트 3개와 영상 흐름이 포함되었는가?
- [ ] 영상 흐름에 실제 사용 가능한 예시 멘트/썸네일 텍스트가 있는가?
- [ ] 플랫폼별 특성이 반영되었는가?
- [ ] 국가별 문화적 특성이 반영되었는가?
- [ ] 리워드 수준에 맞는 가이드라인 엄격성이 설정되었는가?

| 오류 상황 | 대응 |
